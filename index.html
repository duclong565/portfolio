sydes.sandbox

Self-initiated · 2026 · engine + UI

A drag-and-drop system-design sandbox: draw a service topology on a canvas, press Run, and watch real Docker containers spin up under k6 load with per-service status and logs. A graph compiler translates the typed diagram (Service / Kafka / Worker / DB / Load Balancer nodes) into a docker-compose.yml plus nginx upstream config and a k6 script, inferring edge semantics — publish / subscribe / round-robin / DSN — from node types. Each experiment runs on its own isolated Docker bridge network and exercises a Saga chain (service → Kafka → worker → Postgres) end-to-end with Kafka consumer groups on KRaft. A local Fastify agent wraps the engine over HTTP/JSON for a React + React Flow browser canvas; seam/dependency-injection throughout keeps the whole engine unit-testable without Docker. Live page · source.
